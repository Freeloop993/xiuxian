## 三、剧情与地图绑定蓝图（保证逻辑自洽）

## 1) 世界骨架（当前MVP只做人界）
- 位面：人界
- 区域节点：
  - 青石村（出生与恢复）
  - 天南坊市（交易与情报）
  - 宗门驻地（突破许可与任务）
  - 乱星海岸（挂机与材料）
  - 古修遗府（主线冲突与重大机缘）

## 2) 节点功能规则（避免“地图只是装饰”）
- 青石村：安全区，低收益，高恢复，适合新手稳境。
- 天南坊市：资源区，可购买突破材料、触发传闻。
- 宗门驻地：规则区，接委托、拿许可、推进势力关系。
- 乱星海岸：挂机优先区，稳定收益但有小概率受伤事件。
- 古修遗府：剧情区，随世界事件阶段变化，风险最高，收益最高。

## 3) 主线阶段（与事件衍化系统一致）
- 萌芽期：坊市出现“遗府禁制微光”传闻。
- 发展期：宗门与散修都在抢线索，地图上遗府点位高亮。
- 激化期：遗府外围冲突，玩家需在“宗门协助”与“散修潜入”间抉择。
- 终末期：遗府禁制破碎，触发关键战与核心奖励。
- 余波期：势力关系重算，开放新支线或平稳修炼窗口。

## 4) 行动到结果的闭环（剧情通畅关键）
- 玩家点击地点 -> 生成“地点动作指令”（如：前往天南坊市）
- 引擎根据地点+境界+事件阶段计算结果
- 返回 `structured_turn`（effects/world_changes）
- LLM按结构化结果叙事
- 地图状态更新（当前地点、可达点、风险提示）

这样每一步都有“输入-规则-反馈-新目标”，不会变成无意义随机文本。

## 四、地图剧情的最小可玩循环（上线即能玩）
- 循环1：青石村闭关 -> 坊市买丹 -> 宗门接任务 -> 海岸挂机
- 循环2：坊市拿遗府线索 -> 宗门申请支援 -> 遗府探索
- 循环3：遗府冲突结算 -> 回村恢复 -> 新阶段再开

## 五、你出图时的统一约束
- 全部使用中文提示词。
- 图像里不出现任何文字、数字、Logo、水印。
- 地图类用 16:9；图标类用 1:1 透明底。
- 风格统一：像素风 + 修仙古风，不要写实照片风。
